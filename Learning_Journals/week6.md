# Week 06 Learning Journal

### Learning Activities & Resources

* Setting up group collaboration infrastructure and workspace for the team web project.
* Joining the group GitHub repository.
* Integrating GitHub repository updates and Trello project board alerts into the private group Slack channel using commands.
* Connecting with all team members on LinkedIn to establish professional networks.
* Researching and testing initial project planning tasks, such as evaluating web hosting platforms.

---

### Estimated hours of Explicit Learning Activity

Around 2 hours

---

### Content Insights

* Setting up integrated team workflows early establishes clear lines of communication and transparent tracking across the entire project lifecycle.
* Integrating GitHub with Slack provides real-time notifications for commits, pushes, and pull requests, keeping all team members automatically informed without requiring manual status updates.
* Formulating a comprehensive Group Agreement defines clear expectations for team communication channels, response times, conflict resolution, and technical skill allocation.
* Project planning boards (such as Trello or GitHub Projects) enable breaking down broad project goals into manageable, trackable tasks assigned to individual team members.

---

### Career/Employability/Learning Insights

* Collaborative tools like Slack, GitHub, and agile project management boards are industry standards across modern software engineering and technical teams.
* Establishing clear team agreements and communication protocols mirrors real-world Agile software development and project management environments.
* Building professional networks on LinkedIn and learning to coordinate effectively with team members develops crucial soft skills like accountability, communication, and teamwork.
* Understanding workflow automation—such as linking repository triggers directly to team communication channels—increases operational efficiency and developer transparency in group environments.
